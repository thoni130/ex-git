Présentation de GIT

Conçu en 2005 par Linus Torvalds, GIT est un logiciel de gestion de version qui enregistre l'évolution d'un fichier ou d'un ensemble de fichiers, afin d'avoir une trace antérieur de ses données, tout en rapidité et efficacité.

Par ailleurs, ils générent tout de méme des avantages et des inconvenients.

  • Avantages :
  1. Gestion de branches: possibilité de travailler sur plusieurs projets sans se marcher sur les pieds.
  2. La rapidité: lorsque la mise à jour est effectuée, les données sont empaquetées et compréssées rapidement.
  • Inconvénients :
  1. Complexité: ce n'est pas un outil simple à manipuler, trés compliqué pour les débutants car il y a un trés grand nombre de commande.

Origine du nom

De son créateur Linus Torvalds la signification de GIT est particulière, il donne un nom à ses projets en fonction de sa personnalité. Pour cela il a surnommé «conn*rd» de la langue argot britannique qui représente son narcissisme et son coté égocentrique.

Quelques commandes :

Git dispose notamment des commandes suivantes:

  1. git init : crée un nouveau dépôt.
  2. git clone​ : clone un dépôt distant.
  3. git add​ : ajoute de nouveaux objets blobs dans la base des objets pour chaque fichier modifié depuis le derniercommit les objets précédents restent inchangés.
  4. git commit :​ intègre la somme de contrôle d'un objet tree et les sommes de contrôle des objets commits parents pour créer un nouvel objet commit
  5. git branch :​ crée une nouvelle branche de développement.
  6. git merge :​ fusionne plusieurs branches de développement.
